Для начала, нам нужно решить задачу аналитически.

$$-u''(x) + u(x) = x,\;\;0 \leqslant x \leqslant 1,\;\;u(0) = u(1) = 0$$

Рассмотрим соответствующее однородное уравнение.

$$-u''(x) + u(x) = 0$$$$x^2 - 1 = 0$$

Корни $x \in \{-1, 1\}$, оба кратности один.

Тогда общее решение имеет вид $$c_1 e^{x} + c_2 e^{-x}$$

Возвращаемся к исходному неоднородному.

Рассмотрим частное решение $u(x) = x$ неоднородного уравнения

$$-u''(x) + u(x) = x$$

Мы можем получть общее решение неоднородного урванения как сумму общего решения однородного и частного решения неоднородного, тем самым общее решение неоднородного имеет вид:

$$c_1 e^{x} + c_2 e^{-x} + x$$

Осталось подобрать константы (подставим точки x = 0, x = 1):

\begin{equation*} \begin{cases} c_1 + c_2 = 0 \\ c_1 e + \frac{c_2}{e} = -1 \end{cases} \Leftrightarrow \begin{cases} c_2 = -c_1 \\ c_1 e + \frac{c_2}{e} = -1 \end{cases} \Leftrightarrow \begin{cases} c_1 = \frac{1}{\frac{1}{e} - e} \\ c_2 = \frac{-1}{\frac{1}{e} - e} \end{cases} \end{equation*}

Итого:

$$u(x) = \frac{1}{\frac{1}{e} - e} \cdot (e^x - e^{-x}) + x = \frac{e \cdot (e^{x} - e^{-x})}{1 - e^2} + x$$

Теперь опишем функцию для нахождения точного решения:

import numpy as np

def get_u(x):
    return x + (np.exp(1 + x) - np.exp(1 - x)) / (1 - np.exp(2))

def get_A(N):
    h = 1.0 / (N + 1)
    A = []
    for i in range(N):
        current = []
        for j in range(i - 1):
            current.append(0)
        if i > 0:
            current.append(-h**(-2))
        current.append(2 * h**(-2) + 1)
        if i < N - 1:
            current.append(-h**(-2))
        for j in range(N - i - 2):
            current.append(0)
        A.append(current)
    return np.array(A)

               plot_labels=None, color_shapes=None, points=None):

Для начала, заметим, что в нашем случае

$$2h^{-2} + 1 = |c_i| > |a_i| + |b_i| = 2h^2,$$

то есть выполняется достаточное условие разрешимости прогонки.

def tridiagonal_algo_solver(a, b, c, d):
    N = len(a)
    alphas = [b[0] / c[0]]
    betas = [d[0] / c[0]]
    for i in range(N - 1):
        alphas.append(b[i + 1] / (c[i + 1] - alphas[-1] * a[i + 1]))
        betas.append((d[i + 1] + betas[-1] * a[i + 1]) / (c[i + 1] - alphas[-1] * a[i + 1]))
    ans = [0 for _ in range(N)]
    ans[N - 1] = betas[-1]
    for i in range(N - 2, -1, -1):
        ans[i] = ans[i + 1] * alphas[i] + betas[i]
    return np.array(ans)

Как несложно заметить, прогонка работает быстрее. Это можня связать например с тем, что прогонка явно использует, что матрица имеет вполне определенный вид (трехдиагональный) и не только ассимптотически лучше (сложность алгоритма по времени $\mathcal{O}(N)$), но что важно совершает меньше операций с вещественными числами (линейное число). Также прогонка требует и по памяти лишь линейное количество затрат. numpy вызывает один из методов LAPACK, этот метод не оптимизирован для работы с такими матрицами. Число операций может быть порядка $N^2$ (против нашего линейного).

Вообще, scipy умеет работать с разряженными матрицами, а для решения уравнений с его помощью предлагается использовать scipy.linalg.solve_banded

from scipy.linalg import solve_banded

def get_ab(A, N):
    ab = [[0 for _ in range(N)] for _ in range(3)]
    
    for i in range(N):
        if i != 0:
            ab[0][i] = A[i - 1][i]
        ab[1][i] = A[i][i]
        if i != N - 1:
            ab[2][i] = A[i + 1][i]
    
    return np.array(ab)

def get_u_vector_banded_solver(N):
    A = get_A(N)
    ab = get_ab(A, N)
    b = get_b(N)
    return solve_banded((1, 1), ab, b)

def SOR(A, b, N, iter_number, w):
    xs_intermed = []
    xs = [0 for _ in range(N)]
    for _ in range(iter_number):
        xs_new = []
        for i in range(N):
            xs_new.append((1 - w) * xs[i] + w * (b[i] -
                                                sum([A[i][j] * xs_new[j] for j in range(i)]) -
                                                sum([A[i][j] * xs[j] for j in range(i + 1, N)])) / A[i][i])
        xs_intermed.append(xs_new)
        xs = [x for x in xs_new]
    return xs_intermed
